# m3u8-tui 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位

**m3u8-tui** 是一个基于 `m3u8-downloader` 命令行工具的 TUI（Terminal User Interface）终端界面程序，提供可视化的多任务下载管理能力。

### 1.2 目标用户

需要通过 M3U8 链接批量下载视频、管理并发下载任务的用户。

### 1.3 核心价值

- 将单一命令行工具包装为多任务、可视化的终端界面
- 支持同时运行多个下载实例，每个实例独立配置参数
- 实时展示每个任务的下载进度和信息
- 断点续管：退出再进入时恢复任务状态
- 可视化配置管理

---

## 2. 技术选型

| 项目 | 方案 |
|------|------|
| 语言 | Go |
| TUI 框架 | [Bubble Tea](https://github.com/charmbracelet/bubbletea) |
| 组件库 | [Bubbles](https://github.com/charmbracelet/bubbles) (viewport, progressbar, textinput 等) |
| 布局组件 | [Lip Gloss](https://github.com/charmbracelet/lipgloss) |
| 数据持久化 | JSON 文件 (`~/.config/m3u8-tui/state.json`) |
| 配置管理 | 解析项目根目录 `config.json`，支持运行时修改 |

**选型理由：**

- `m3u8-downloader` 本身是 Go 编写的，使用 Go 生态可无缝管理子进程
- Bubble Tea 是 Go 生态最成熟的 TUI 框架，基于 Elm Architecture 模型
- Lip Gloss 提供声明式样式定义，适合构建复杂的终端布局

- `↑`（或 `k`）：在当前区域内向上选择；已在第一项时，跳转到上一个区域
- 底部功能栏始终显示当前焦点区域和可用操作

### 3.2 标签页切换

- `Tab` 键在两个标签页之间切换
- `1` 键直接跳转到任务面板，`2` 键直接跳转到配置面板
- 当前标签页标签以高亮（`●` 指示器）标识
- 底部功能栏随当前标签页自动更新，显示可用的快捷键提示

### 3.3 任务面板内焦点流动

任务面板内部按三个焦点区域排列：**Input → Active → History**。

焦点通过 `↑`/`↓`（或 `j`/`k`）在区域内选择和跨区域流动：

- 焦点在 **Input** 时，`↓` 进入 Active（选中第一个任务）；`↑` 无操作
- 焦点在 **Active** 时，`↓` 到最后一项继续 `↓` 进入 History；`↑` 到第一项继续 `↑` 回到 Input
- 焦点在 **History** 时，`↓` 无操作（已到底）；`↑` 到第一项继续 `↑` 回到 Active
- 每个区域内部可独立滚动，超出显示高度的任务通过方向键滚动浏览
- `Enter` 在当前区域执行确认操作（启动任务 / 查看详情 / 展开日志）

## 4. 功能需求

### 4.1 参数输入与任务启动 (FR-01)

**描述：** 用户在输入框中整条输入 `m3u8-downloader` 的参数，按 `Enter` 启动下载任务。

**输入格式示例：**

```
-u "https://surrit.com/b5d2679b-9675-48f3-a16e-8ba649222fe7/720p/video.m3u8" -n 32 -H "Referer: https://missav.ai/cn/sone-852-uncensored-leak" -o "[missav.ai] sone-852-uncensored-leak_1280x720"
```

**详细规则：**

1. 输入框位于任务面板顶部，接受完整的命令行参数字符串
2. 支持所有 `m3u8-downloader` 参数：`-url`, `-u`, `-name`, `-o`, `-dir`, `-d`, `-threads`, `-n`, `-retries`, `-r`, `-force`, `-f`, `-keep`, `-insecure`, `-k`, `-cookie`, `-c`, `-header`, `-H`, `-proxy`, `-p`
3. 启动前校验：至少需要 `-url` 或 `-u` 参数
4. 默认参数填充：如果未指定 `-n`/`-threads`，自动使用 `config.json` 中的 `default_threads`；未指定 `-dir`/`-d`，使用 `config.json` 中的 `default_dir`（若配置为空则使用当前目录）
5. 回车启动后，输入框清空，任务出现在"活动任务"列表顶部
6. 成功启动后自动聚焦到新任务

**参数解析方式：**

使用支持 shell 引号规则的解析器（如 `github.com/google/shlex` 库的 `shlex.Split` 函数）将用户输入的参数字符串正确分割为 `[]string`，传递给 `exec.Command("m3u8-downloader", args...)`。

> 需要注意：不能使用简单的 `strings.Split(input, " ")` 按空格分割，否则包含空格引号包裹的参数（如 `-o "my video.mp4"`）会被错误拆碎。`shlex.Split` 遵循 POSIX shell 的引号规则，其解析结果与 shell 脚本中 `"$@"` 的行为一致。

### 4.2 多任务并发管理 (FR-02)

**描述：** 支持同时运行多个 `m3u8-downloader` 实例，每个实例独立管理。

**详细规则：**

1. 无并发任务数上限（由系统资源决定）
2. 每个任务拥有唯一 ID（UUID v4）
3. 每个任务维护独立的状态机：

```
   pending → running → completed
                     → failed
                     → cancelled
```

4. 任务数据结构：

```json
{
  "id": "uuid-v4",
  "args": "用户输入的原始参数字符串",
  "status": "running",
  "pid": 12345,
  "start_time": "2026-06-16T18:34:18+08:00",
  "end_time": null,
  "output_log": ["line1", "line2", "..."],
  "progress": {
    "percent": 45.2,
    "downloaded": 320,
    "total": 708,
    "speed": "12.3 MB/s",
    "eta": "00:02:15"
  }
}
```

> **字段说明：** `percent` 为浮点数百分比 (0-100)；`downloaded` 和 `total` 单位为 TS 片段数 (fragments)；`speed` 为字符串（含数值和单位，如 `"12.3 MB/s"`）；`eta` 为字符串格式的剩余时间。

---

### 4.3 实时输出与进度展示 (FR-03)

**描述：** 实时捕获每个 `m3u8-downloader` 子进程的 stdout/stderr 输出，解析进度信息并展示。

**详细规则：**

1. **输出捕获：** 通过 `exec.Command` 的 `StdoutPipe` 和 `StderrPipe` 实时读取子进程输出，逐行推送到对应任务的 `output_log`
2. **进度解析：** `m3u8-downloader` 的进度输出有固定格式，通过正则匹配提取：
   - 百分比: `(\d+\.?\d*)%`
   - 下载/总数: `(\d+)/(\d+)`
   - 速度: 匹配速度字段
   - ETA: 匹配剩余时间字段

3. **活动任务卡片展示：**

```
▶ [#1] 运行中  [████████░░░░░░░░░░░░] 45%  |  320/708  |  12.3 MB/s  |  ETA 00:02:15
  args: -u "https://..." -n 32 -o "video"
  [14:30:01] Downloading ts fragments...
  [14:30:02] Fragment 320/708 completed
```

4. **实时刷新机制：**
   - 子进程每行输出触发 Bubble Tea `tea.Cmd` 消息
   - 进度更新频率受控（最小更新间隔 200ms），避免性能问题
   - 使用 `tea.Tick` 在无输出时保持渲染

5. **日志滚动：** 每个任务卡片内显示最近 N 行（默认 3 行）输出日志，通过方向键可展开查看更多

---

### 4.4 任务完成与历史归档 (FR-04)

**描述：** `m3u8-downloader` 进程结束后，任务自动转入历史列表。

**判断进程结束的方式：**

1. **主要方式：** 子进程 `Wait()` 调用返回，表示进程退出
2. **退出码判断：**
   - 退出码 0 → 状态为 `completed`
   - 退出码非 0 → 状态为 `failed`
3. **额外判断：** 解析输出日志，如果出现特定完成标志（如 "Download completed", "下载完成" 等关键词），辅助确认为成功

**历史任务展示：**

```
┌─ 历史任务 (3) ──────────────────────┐
│ ✓ [#3] 已完成  100%   00:12:34      │
│   args: -u "https://..." -o "video1" │
│ ✗ [#4] 失败            00:00:45      │
│   args: -u "https://..." -o "video2" │
│   错误: connection timeout           │
│ ✓ [#5] 已完成  100%   00:08:12      │
└──────────────────────────────────────┘
```

4. 历史任务日志保留完整输出，可通过 `Enter` 查看详情
5. 用户手动使用 `Ctrl+C` 取消任务 → 状态为 `cancelled`（注意区分：一次 Ctrl+C 取消选中的任务，两次 Ctrl+C 退出程序）

### 4.5 会话持久化 (FR-05)

**描述：** 退出 TUI 界面后再次进入，能够继续显示还在执行中的任务状态。

**详细规则：**

1. **持久化文件：** `~/.config/m3u8-tui/state.json`
2. **持久化时机：**
   - 每 2 秒自动保存一次状态快照
   - 正常退出时（`Ctrl+C` 或 `q` 键）立即保存
3. **持久化内容：**

```json
{
  "version": 1,
  "tasks": [
    {
      "id": "uuid",
      "args": "...",
      "status": "running",
      "pid": 12345,
      "start_time": "...",
      "history": []
    }
  ],
  "history": [
    {
      "id": "uuid",
      "args": "...",
      "status": "completed",
      "start_time": "...",
      "end_time": "...",
      "output_log": ["..."]
    }
  ]
}
```

4. **恢复机制：**
   - 启动时读取 `state.json`
   - 根据 PID 检查进程是否仍然存活（`syscall.Kill(pid, 0)`）
   - **存活** → 重新关联 stdout/stderr pipe 继续捕获输出（注：对于已脱离的进程只能通过 `/proc/{pid}/fd` 访问，或者将子进程设计为以 nohup 模式运行并重定向到日志文件，TUI 启动时通过读取日志文件恢复状态）
   - **不存活** → 标记为 `cancelled` 或通过解析已有输出判断终态

**进程存活恢复的设计选择：**

由于重新关联已运行进程的 stdout pipe 存在技术限制，推荐采用以下方案：

- **方案A（推荐）：** 子进程启动时 `m3u8-downloader` 的输出同时重定向到 `~/.config/m3u8-tui/logs/{task_id}.log`，TUI 始终通过读取日志文件来展示输出。PID 仅用于判断进程存活状态和发送信号。
- **方案B：** 退出时 kill 所有子进程并记录为 interrupted，下次进入时重新启动（简单但体验差）

本 PRD 推荐**方案A**。

### 4.6 config.json 配置管理 (FR-06)

**描述：** 在配置面板标签页中展示 `config.json` 的默认配置信息，并支持修改。

**配置项对应关系：**

| config.json 字段 | 说明 | 对应 m3u8-downloader 参数 |
|------------------|------|---------------------------|
| `default_threads` | 默认线程数 | `-n` / `-threads` |
| `timeout_sec` | 超时秒数 | （内部参数） |
| `max_retries` | 最大重试次数 | `-r` / `-retries` |
| `proxy` | 代理地址 | `-p` / `-proxy` |
| `insecure` | 跳过HTTPS验证 | `-k` / `-insecure` |
| `default_dir` | 默认保存目录 | `-d` / `-dir` |
| `user_agents` | UA 列表数组 | （内部参数） |

**功能细则：**

1. **配置面板标签页** 以表单形式展示所有配置项和当前值
2. **编辑模式：** 切换到配置标签页后，按 `e` 进入编辑模式
   - 使用方向键选择配置项
   - `Enter` 开始编辑当前值
   - `Esc` 取消编辑，`Enter` 确认修改
3. **修改即时生效：** 修改后立即写入 `config.json`，并影响之后启动的新任务（不影响正在运行的任务）
4. **重置默认值：** 提供 `Ctrl+R` 快捷键将配置恢复为程序内置默认值
5. **User-Agents 数组：**
   - 显示为多行列表
   - 支持 `a` 添加一行，`d` 删除当前选中行

## 6. 数据流与架构

### 6.1 Elm 架构（Bubble Tea）

```
                    ┌──────────┐
                    │   Model  │  ← 全局状态（任务列表、配置、焦点等）
                    └────┬─────┘
                         │
              ┌──────────┼──────────┐
              ▼          ▼          ▼
         ┌────────┐ ┌────────┐ ┌────────┐
         │  Init  │ │ Update │ │  View  │
         └───┬────┘ └───┬────┘ └───┬────┘
             │           │           │
             │    ┌──────▼──────┐    │
             │    │   Cmd      │    │
             │    │ (副作用)    │    │
             │    └──────┬─────┘    │
             │           │           │
             └───────────┴───────────┘
                         │
                         ▼
                   终端渲染输出
```

### 6.2 消息类型

```go
type Msg interface{}

// 子进程输出消息
type TaskOutputMsg struct {
    TaskID string
    Line   string
}

// 子进程退出消息
type TaskExitMsg struct {
    TaskID   string
    ExitCode int
    Err      error
}

// 进度更新消息
type ProgressUpdateMsg struct {
    TaskID   string
    Percent  float64
    Done     int
    Total    int
}

// 定时刷新 (UI tick)
type TickMsg time.Time

// 自动保存
type AutoSaveMsg struct{}

// 用户输入
type KeyMsg tea.KeyMsg
```

## 7. 命令行参数

m3u8-tui 本身支持以下命令行参数：

```
m3u8-tui [OPTIONS]

Options:
  -h, --help            显示帮助信息
  -v, --version         显示版本号
  -c, --config PATH     指定 config.json 路径（默认: 当前目录/config.json）
  --state-dir PATH      指定状态存储目录（默认: ~/.config/m3u8-tui/）
```

## 8. 非功能性需求

### 8.1 性能要求

- TUI 界面渲染帧率不低于 30fps
- 单个子进程输出解析延迟 < 100ms
- 支持同时运行 20+ 个子进程无明显卡顿
- 日志文件单个任务不超过 10MB（超过自动轮转）

### 8.2 兼容性

- Linux (主要目标平台)
- macOS
- 终端模拟器：支持 xterm-256color 及以上
- 终端最小尺寸：120×40

### 8.3 错误处理

- `m3u8-downloader` 未安装 → 显示错误提示并退出
- `config.json` 缺失 → 使用内置默认配置并自动生成
- `state.json` 损坏 → 丢弃并用空状态启动
- 子进程异常退出 → 标记为 `failed`，记录错误信息
